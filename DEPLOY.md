# Deploy FastX for Free

Host the full stack at **$0/month** using:

| Layer | Service | Free tier |
|-------|---------|-----------|
| Frontend (Angular) | [Netlify](https://netlify.com) | 100 GB bandwidth |
| Backend (Flask) | [Render](https://render.com) | 750 hrs/month (sleeps when idle) |
| Database (MySQL) | [Aiven](https://aiven.io/free-mysql-database) | 1 GB storage, free forever |

---

## Step 1 — Free MySQL on Aiven

1. Sign up at [aiven.io](https://aiven.io) (GitHub login works).
2. **Create service** → choose **MySQL** → pick the **Free** plan → select a region.
3. Wait for the service to start, then open **Overview** and note:
   - Host
   - Port
   - User
   - Password
   - Database name (defaultdb)
4. Download **CA certificate** (Connection info → CA certificate).
   - You will paste this PEM content into Render as `DB_SSL_CA`.

---

## Step 2 — Backend on Render

1. Sign up at [render.com](https://render.com) and connect your GitHub account.
2. **New → Blueprint** → select repo `Bus-Ticket-Booking-App`.
   - Render reads `render.yaml` from the repo root.
   - Or manually: **New Web Service** → repo → set **Root Directory** to `backend`.
3. Set environment variables on the Render service:

| Variable | Value |
|----------|-------|
| `DB_HOST` | Aiven MySQL host |
| `DB_PORT` | Aiven port (usually 12345+) |
| `DB_USER` | Aiven username |
| `DB_PASS` | Aiven password |
| `DB_NAME` | `defaultdb` (or your Aiven DB name) |
| `DB_SSL_CA` | Paste full CA certificate PEM from Aiven |
| `FRONTEND_URL` | Your Netlify URL (set after Step 3, e.g. `https://fastx.netlify.app`) |
| `JWT_SECRET` | Auto-generated by Render, or any long random string |

4. Deploy and copy your API URL, e.g. `https://fastx-api.onrender.com`.
5. Test: open `https://YOUR-API.onrender.com/health` — should return `{"status":"healthy"}`.

> **Note:** Render free tier sleeps after ~15 min idle. First request after sleep takes ~30–60 seconds.

---

## Step 3 — Frontend on Netlify

1. Sign up at [netlify.com](https://netlify.com) and connect GitHub.
2. **Add new site → Import an existing project** → select `Bus-Ticket-Booking-App`.
3. Build settings (auto-detected from `netlify.toml`):
   - Base directory: `frontend`
   - Build command: `npm ci && npm run build`
   - Publish directory: `frontend/dist/frontend/browser`
4. Before deploying, edit `netlify.toml` and `frontend/public/_redirects`:
   - Replace `REPLACE-WITH-YOUR-RENDER-URL` with your Render hostname (no `https://`).
   - Example: `fastx-api.onrender.com`
5. Deploy. Your site URL will be like `https://random-name.netlify.app`.
6. Go back to Render and set `FRONTEND_URL` to your Netlify URL, then redeploy the backend.

2. Login as admin: `admin@fastx.com` / `admin123`

## Troubleshooting

**API returns 503 on /health**
- Check Aiven credentials and `DB_SSL_CA` on Render.
- Ensure Aiven service is running (free tier may pause when inactive — wake it in the Aiven console).

**CORS errors in browser**
- Set `FRONTEND_URL` on Render to your exact Netlify URL (with `https://`, no trailing slash).

**Slow first load**
- Render free tier cold start — normal after idle period.

**Netlify API calls fail**
- Confirm `_redirects` and `netlify.toml` point to the correct Render URL.
- Redeploy Netlify after editing redirect rules.

**Database empty**
- Backend runs `init_db()` on startup and seeds routes automatically on first connect.

## Optional: custom domain

- **Netlify:** Site settings → Domain management → add your domain (free SSL).
- **Render:** Settings → Custom Domains on the web service.

Update `FRONTEND_URL` on Render if you add a custom domain.
